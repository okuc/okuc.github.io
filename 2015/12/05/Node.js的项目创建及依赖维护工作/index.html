Node.js的项目创建及依赖维护工作

要编写nodejs项目，则一般使用向导生成工程样板，然后修改工程样板，达到自已的目地，也有的直接编写各类文件。本人将自已常的生成工程样板的方式在此分享一下，高手请直接飘过。。。。。
在此假定我们node及npm环境已经安装配置完成，可以直接使用。

###使用npm向导创建nodejs样板工程

  1. 创建一目录nodejs-demo，并在命令行中切换到该目录下。输入npm init，按要求输入工程名称、版本号、描述、入口等就可以了。一般直接使用默认就可以，但是有两点，入口现在一般添bin/www,关键词直接输入项目关键字或不输都可以。
  2. 使用命令yum install -g bower安装bower。
  3. 安装完成后，在命令行中，使用bower init,如果在使用的root用户，则在使用初始化项目命令时，要添加--allow-root，即命令为：bower init --allow-root,同样输入一系列问题的答案，即可。在选择模块规范时，可以直接使用默认的amd即可。
  4. bower默认把项目依赖放到bower_components下，我们可以把它更改到其它目录下。如更改到依赖的第三方包到public\lib下，在项目根目录创建一文件.bowerrc，内容如下：
    {
    "directory":"public/lib"
}
  5. 在编写前台所依赖的包时，可直接使用bower来安装，如bower install --allow-root --save bootstrap。
  6. 到此就结束了，但是很可惜，你会发现npm init就生成了一个package.json文件，所以，我更常用的，是下一种方式。

###使用express向导创建nodejs样板工程

  1. 在做web开发时，一般使用express，所以，一般使用express自带的向导来生成样板工程。首先，进行安装express及express-generator两个模块，命令如下：
    npm install -g express
npm install -g express-generator
  2. 使用express example来创建样例工程。
  3. 切换到生成的example工程目录下，使用npm install来下载安装样板工程所依赖的包模块。
  4. 使用node bin/www启动服务，打开浏览器，访问http://localhost:3000,即可成功访问。
  5. 如果要添加后端依赖的模块，如jquery，则直接使用npm安装：npm install -g jquery --save,save参数是为了将该模块添加到package.json配置文件中。
  6. 添加bower，进行前端工程依赖包的管理。首先使用命令yum install -g bower安装bower。
  7. 安装完成后，在命令行中，使用bower init,如果在使用的root用户，则在使用初始化项目命令时，要添加--allow-root，即命令为：bower init --allow-root,同样输入一系列问题的答案，即可。在选择模块规范时，可以直接使用默认的amd即可。
  8. bower默认把项目依赖放到bower_components下，我们可以把它更改到其它目录下。如更改到依赖的第三方包到public\lib下，在项目根目录创建一文件.bowerrc，内容如下：
    {
    "directory":"public/lib"
}
  9. 在编写前台所依赖的包时，可直接使用bower来安装，如bower install --allow-root --save bootstrap。
