《Bootstrap用户手册》学习笔记(一)

第1章 Bootstrap提供的网站框架

  1. Bootstrap中有几种默认样式需要注意，它们专门针对排版和链接。

    • 从body中去掉了margin,这样内容就会紧贴浏览器窗口边缘。
    • 为body应用了background-color:white;。
    • Bootstrap以@baseFontFamily、@baseFontSize、@baseLineHeight属性作为排版的基准，确保整个网站的标题及其他内容的行高一致。

    • Bootstrap通过@linkColor设置全局链接颜色，只在:hover状态下给链接添加下划线。

      要改变默认颜色，可以直接修改相应.less文件中的全局变量值。要么修改scaffolding.less文件，要么修改你自已的样式表。

  2. Bootstrap默认的网格布局包含12列，940像素宽，不支持响应式布局。加载响应式CSS文件后，网格布局会根据视口(viewport)宽度在724像素之间伸缩。视口宽度小于767像素时，说明是平板电脑或更小的设备，布局中的列会垂直堆叠起来。默认宽度下，每个列宽60像素，且向左移20像素。

  3. Bootstrap中，表示行的div样式为.row,作为列的div添加横跨多少列的.span*类。因为共12列，所以.span*中的*加起来必须等于12。这样，可以设计出：3-6-3，4-8，3-5-4，2-8-2等布局。下面是一个例子：

    <div class="row">
  <div class="span8">...</div>
  <div class="span4">...</div>
</div>

  4. 使用.offset*可以向右平移列，*用于指定平移的列数。如，向右平移2列：

    <div class="row">
  <div class="span2">...</div>
  <div class="span7 offset2">...</div>
</div>

  5. 嵌套列，只要在相应的.span*中再增加一个.row，然后在其中包含与父容器列数相等的.span*即可。

    <div class="row">
  <div class="span9">
    Level 1 of column
    <div class="row">
      <div class="span6">Level 2</div>
      <div class="span3">Level 2</div>
    </div>
  </div>
</div>

  6. 流式网格系统的列宽定义使用百分比，而不是像素。把行的样式.row改为.row-fluid，这一行就会成为流式的。下面是两个例子：

    <div class="row-fluid">
  <div class="span4">...</div>
  <div class="span8">...</div>
</div>
<div class="row-fluid">
  <div class="span4">...</div>
  <div class="span4 offset2">...</div>
</div>

流式网格中的嵌套有点不一样，原因是在使用百分比的情况下，每个`.row`都会把列数重置为`12`.举个例子，要想在一个`.span8`中嵌套两个等宽的列，不能使用两个`.span4`,而应该使用两个类为`.span6`的`div`。这是为保证内容具有响应性，因为我们希望内容`100%`填满容器：

    <div class="row-fluid">
      <div class="span8">
                    <div class="row">
                            <div class="span6">...</div>
                            <div class="span6">...</div>
                    </div>
      </div>
    </div>

  1. 要在页面中实理固定宽度、居中的布局，只要把内容统统收到<div class="container">...</div>中即可。既想实现流式布局，又想把所有内容都封装到一个容器里，可以使用<div class="container-fluid">...</div>。流式布局适合很多应用、管理界面及其他项目。

  2. 要让Bootstrap支持响应式布局，必须在<head>标签中添加一个<meta>标签。另外，如果你下载的文件中没有响应式css文件，也要单独下载。例子如下：

    <!DOCTYPE html>
<html>
        <head>
              <title>My amazing Bootstrap site!</title>
              <meta name="viewport" content="width=device-width, initial-scale=1.0">
              <link href="/css/bootstrap.css" rel="stylesheet">
              <link href="/css/bootstrap-responsive.css" rel="stylesheet">
        </head>

  1. 根据浏览器视口的宽度，Bootstrap可以按照纵横比或宽度的范围来优化布局，但最主要还是使用min-width和max-width属性。

    Bootstrap 内部支持 5 种不同的布局，都依赖于CSS 媒体查询。最大布局中每一列的宽度为 70像素，而常规布局中每一列宽度是 60 像素。在适合平板电脑的布局中，列宽缩小为 42像素，再窄一点，每一列就会流动起来，变成在垂直方向上堆叠，并且都与视口同宽。如下

    • 大屏幕：至少1200px，列宽为70px，列间距为30px
    • 常规：至少980px,，列宽为60px，列间距为20px
    • 平板竖屏：至少768px，列宽为42px，列间距为20px
    • 手机到平板：至少767px，流式列，宽度不固定
    • 手机：至多480px，流式列，宽度不固定

  2. 要根据媒体查询添加自定义的 CSS ，可以像下面这样把所有规则写到一个 CSS 文件中，也可以单写一个 CSS 文件：

    /* 大屏幕 */
@media (min-width: 1200px) { ... }
/* 平板竖屏到横屏到常规屏幕 */
@media (min-width: 768px) and (max-width: 979px) { ... }
/* 手机横屏到平板竖屏 */
@media (max-width: 767px) { ... }
/* 手机横屏及更小屏幕 */
@media (max-width: 480px) { ... }

  1. 对于大型网站，应该把不同的媒体查询分别保存在单独的 CSS 文件中。而在 HTML的<link>标签中，可以根据条件加载它们。这样可以保证加载的文件相对小，但在响应式布局中会增加 HTTP 请示。如果你使用 LESS 编译 CSS ，可以把它们都放到一个文件中：



    <link rel=”stylesheet” media=”(min-width: 768px) and (max-width: 979px)”

    href="tablet.css" />

  2. Bootstrap为响应式开发提供了一个辅助类，但是尽量少用。除非如下情况，如在某些布局中显示或隐藏自定义元素。如在常规布局中可以包含一个页眉，但是移动布局中需要简化它，只显示少量元素，这时就可以组页眉中需要隐藏的元素添加.hidden-phone类

    | 类 | 手机 | 平板 | 电脑 |
    | .visible-phone | 显示 | 隐藏 | 隐藏 |
    | .visible-tablet | 隐藏 | 显示 | 隐藏 |
    | .visible-desktop | 隐藏 | 隐藏 | 显示 |
    | .hidden-phone | 隐藏 | 显示 | 显示 |
    | .hidden-tablet | 显示 | 隐藏 | 显示 |
    | .hidden-desktop | 显示 | 显示 | 隐藏 |

  3. 响应式开发有两种主要的思路。当前备受推崇的方法是从移动版本做起，最后才是桌面网站。 Bootstrap 的思路刚好相反，你可以用它来搭建一个 “ 差不多各平台都能用 ”的桌面网站。

第2章 Bootstrap预定义的CSS样式(1)

  1. 对页面排版，BootStrap默认字体为Helvetica Neue,Helvetica,Arial,sans-serif;。这些字段也是主流操作系统默认支持的标准字体。如果字体不存在，那么sans-serif会告诉浏览器使用默认的字体。页面正文的font-size是14象素，line-height是20象素。所有段落的margin-bottom是10像素,也就是line-height的一半。

  2. 标题：Bootstrap为所有6个级别的标题都定义了样式，其中 <h1>是36像素高，<h6>是12像素高。如果想给这些标题添加行内级的子标题 ，可以使用<small>，结果文本会稍小一些，颜色也会浅一些。以<h1>为例，它的子标题的文本是24像素高，不加精，而且是灰色：

    h1 small {
    font-size:24px;
    font-weight:normal;
    line-height:1;
    color:#999;
    ｝

  3. 正文:强调段落内的文本，可以加上class=”lead”,这会导致段落内文本稍稍变大、变细，行高也变高。例子如下：

    <p class="lead">Bacon ipsum dolor sit amet tri-tip pork loin ball tip frankfurter
swine boudin meatloaf shoulder short ribs cow drumstick beef jowl.
Meatball chicken sausage tail, kielbasa strip steak turducken venison prosciutto.
Chuck filet mignon tri-tip ribeye, flank brisket leberkas. Swine
turducken turkey shank, hamburger beef ribs bresaola pastrami venison rump.</p>

    在正文中也可以使用<small>，这样文本会缩小到原来大小的85%。

  4. 粗体:使用<strong>标签，这样会给相应的文本应用font-weight:bold;。

  5. 斜体:使用<em>标签，它一般用来在文本中强调某些内容。

  6. 强调相关的类：除了上述的<strong>、<em>，Bootstrap还提供了一些表示强调的类，这些类可以应用组<p>或<span>,它们是一些颜色上的变化。

    <p class="muted">This content is muted</p>
<p class="text-warning">This content carries a warning class</p>
<p class="text-error">This content carries an error class</p>
<p class="text-info">This content carries an info class</p>
<p class="text-success">This content carries a success class</p>
<p>This content has <em>emphasis</em>, and can be <strong>bold</strong></p>

  1. 缩写词：HTML的<abbr>元素用于标记缩写词或首字母缩小，如WWW、HTTP。标记了缩写词之后，浏览器、拼写检查器、翻译系统或搜索引擎都据以得知文本的性质。Bootstrap会在<abbr>元素下方添加一条浅色的虚线，并在光标悬停时显示完整的文本内容。

    <abbr title="Real Simple Syndication">RSS</abbr>

    为添加.initialism类可以得到稍小一些的提示文本。如：

    That joke had me <abbr title="rolling on the floor,laughing out loud" >ROTFLOL</abbr>

  2. 地址:在页面中使用<address>元素可以让屏幕阅读器和搜索引擎找到地址或电话号码。这个标签也可以标记电子邮件地址。<address>默认是display:block;，因此必要时可以用<br>标签来换行（例如把城市与街道分开）：

    <address>
  <strong>O'Reilly Media, Inc.</strong><br>
  1005 Gravenstein HWY North<br>
  Sebastopol, CA 95472<br>
  <abbr title="Phone">P:</abbr> 
  <a href="tel:+17078277000">(707) 827-7000</a>
</address>
<address>
  <strong>Jake Spurlock</strong><br>
  <a href="mailto:#">flast@oreilly.com</a>
</address>

  3. 引用:要在页面中标记引用或名人名言的文本块，使用<blockquote>标签。为了保证效果，必要时可以添加<br>换行，用<p>分段。 Bootstrap 默认会缩进引用的文本，并在左侧加一条粗边框。标记引用来源可以用<small>标签，人名可以用<cite>标签：

    <blockquote>
        <p>That this is needed, desperately needed, is indicated by the incredible uptake of Bootstrap. I use it in all the server software I'm working on. And it 
shows through in the templating language I'm developing, so everyone who uses it will find it's "just there" and works, any time you want to do a Bootstrap 
technique. Nothing to do,no libraries to include. It's as if it were part of the hardware. Same approach that Apple took with the Mac OS in 1984.</p>
        <small>Developer of RSS, <cite title="Source Title">Dave Winer</cite>
    </small>
</blockquote>

要想让引用右对齐，可以组`<blockquote>`添加`.pull-right`类。添加这个类后，不光是文本会右对齐，整个引用块都会浮动到右边。

  1. 列表:Bootstrap支持三种列表：有序列表、无序列表和定义列表。

    • 无序列表：无序列表没有特定顺序，一般样式是添加圆点符号。如果想去掉无序列表前的圆点，可以在相应的<ul>标签中添加class="unstyled"

      Favorite Outdoor Activities


      • Backpacking in Yosemite

      • Hiking in Arches

        • Delicate Arch

        • Park Avenue



      • Biking the Flintstones Trail

    • 有序列表：样式上使用数字而不是圆点。适合任务清单、操作步骤、甚至博客的评论列表：

      Self-Referential Task List


      1. Turn off the internet.

      2. Write the book.

      3. … Profit?

    • 定义列表：定义列表没有块级的<li>元素，而是每个列表项都由<dt>和<dd>元素共同组成。<dt>代表definition term（词条），类以词典里收录的词语（短语），<dd>自然就是<dt>的定义。

      标记文本最佳方式，就是使用<dl>，把标题用<dt>,文本用<dd>元素。这样不仅可以分行显示，还可以并排显示。

      Common Electronics Parts


      LED

      A light-emitting diode (LED) is a semiconductor light source.

      Servo

      Servos are small, cheap, mass-produced actuators used for radio
      control and small robotics.

      要让词条和解释并排（显示在一排）显示，只要给<dl>标签加上class="dl-horizontal"取可。

    定义列表在水平布局时，词条如果过长而左边放不下，将被text-overflow属性截短。此外，在视口较窄的时候，词条和解释将自动变成默认垂直堆叠布局。

  2. 代码:在 Bootstrap 中有两种方式显示代码，一种是使用<code>标签，另一种是使用<pre>标签。一般来说，行内代码使用前者，代码段使用后者：

    Instead of always using divs, in HTML5, you can use new elements like
    <section>, <header>, and
    <footer>. The html should look something like this:

         <article>
       <h1>Article Heading</h1>
     </article>

    使用<pre>和<code>标签标记代码时，一定要把代码中的左尖括号和右尖括号替换成&lt;和&gt;。

  3. 表格：Bootstrap支持的表格元素如下所示：

    | 标签 | 作用 |
    |

    | 包装元素，用于显示表列式数据 |
    | | 表头行（）的容器 |
    | | 表格行（）的容器 |
    | | 单元格（
    或）容器 |
    |     		| 普通单元格 |
    |     		| 一行或一列（取决于作用范围和位置）中作为表头的单元格 |
    |
    | 用于包装表格的说明或小结，对屏幕阅读器特别有用 |

    如果你想要一个简单的表格，内容间有少许内边距，同时还有水平的分隔线，只要给<table>添加.table类即可,这种基本样式由几条简单的横线组成。

除了基本的`.table`类，`Bootstrap`还提供了4个不同的样式，分别是：`.table-striped`、`.table-bordered`、`.table-hover`和`.table-condensed`。

-   `.table-striped`，为中的表格行加上条纹背景。这是通过`:nth-child`选择符实现的，但是不支持IE7及IE8。
-   `.table-bordered`类，可以为所有单元格加上边框，为整个表格加上圆角。
-   `.table-hover`:光标悬停的行会带有浅灰色背景。
-   `.table-condensed`：，会减少一半的内边距（每行高度），以紧缩表格。这种样式适合信息密集的情况。

可以添加给表格行的类，用于改变特定的颜色：

-   `.success`：表示成功或积极的情形，绿色。
-   `.error`：表示危险或消极的情形，红色。
-   `.warning`：表示警告，黄色。
-   `.info`：表示代替默认样式，蓝色。
