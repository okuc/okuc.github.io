Linux折腾记(2)-CentOS6下匿名共享可读写文件夹

想把windows上的一些资料定时备份到linux，首先想到的就是利用共享文件夹的方式，折腾了一番，最终还是搞成功了。过程如下：

  1. 首先安装samba及samba-client,中间不管提问什么，一路y即可。
    yum install samba samba-client
  2. 在home下创建文件夹test。
    mkdir /home/test

  3. 将/etc/samba/smb.conf文件中的内容替换成。

    [global]
security=share
dos charset=cp936
unix charset=gb2312

[mygitback]
comment = share
path = /home/test
browseable = yes
guest ok = yes
writable = yes

  4. 测试配置是否正确testparm。

    [root@htzh199 ~]# testparm
Load smb config files from /etc/samba/smb.conf
rlimit_max: increasing rlimit_max (1024) to minimum Windows limit (16384)
Processing section "[mygitback]"
WARNING: The security=share option is deprecated
Loaded services file OK.
Server role: ROLE_STANDALONE
Press enter to see a dump of your service definitions

[global]
    dos charset = cp936
    unix charset = gb2312
    security = SHARE
    idmap config * : backend = tdb

[mygitback]
    comment = share
    path = /home/gitback
    read only = No
    guest ok = Yes
  5. 启动服务。service smb restart
  6. 此时共享可以访问了，在window下直接\\ip即可。不要用win10，用win10仍然没有办法访问。但是使用其它系统，只有读权限，没有写权限，添加写权限：
    vi /etc/selinux/config
# This file controls the state of SELinux on the system.
# SELINUX= can take one of these three values:
#     enforcing - SELinux security policy is enforced.
#     permissive - SELinux prints warnings instead of enforcing.
#     disabled - No SELinux policy is loaded.
SELINUX=disabled  #将这个改为disable
# SELINUXTYPE= can take one of these two values:
#     targeted - Targeted processes are protected,
#     mls - Multi Level Security protection.
SELINUXTYPE=targeted
  7. 再次用windows访问，现在能够正常进行读写了。
  8. 如果还提示无写权限，可以使用chmod 777 /home/test进行授权，然后就可以了。
  9. 可以继续配置随系统启动:chkconfig smb on
