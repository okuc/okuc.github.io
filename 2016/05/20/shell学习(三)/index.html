shell学习(三)

shell中的输入输出重定向

  • 标准输入是键盘，标准输出是终端显示器。
  • 输出重定向
    • command > outputfile //大于号两端要有空格分开
    • command >> outputfile

  • 一个例子

    echo 22222 > ./iotest
echo 111 > ./iotest //会把上一行覆盖掉
echo 111 >> ./iotest //不会覆盖掉原有数据，会新起一行
  • 输入重定向
    • command < outputfile
  • 例子： cat < iotest > iotest2

  • 内联输入重定向

    command << marker
data input
marker

  • 例子：

    result = `bc << EOF
>var1=3
>var2=4
>var3=var1+var2
>print var3
>EOF`

echo $result
  • Linux系统将每个对象当作文件处理，包括输入输出过程。
  • Linux用文件描述符来标识每个文件对象。
  • 文件描述符是一个非负整数，唯一标识会话中打开的文件。
  • 每个过程一次最多可以有9个文件描述符。

  • 文件描述符，缩写，描述

    • 0，STDIN，标准输入
    • 1,STDOUT，标准输出
    • 2，STDERR，标准错误

  • 例子

      ehco hello 1> iotest3
cat 0< iotest
  cat iot 2>iotest4  //错误重定向
  ls io* ff* 1>iotest5 2> errlog //将正确的输出和错误输出进行分离

脚本中的重定向

  • 临时重定向：command >&文件描述符

    #！ /bin/bash

echo "Test Error" >&2
echo "Nomal outPut"

    运行时，直接运行，则都会输出。如果运行时指定的错误输出， ./iotest6 2>> errlog,则只会输出正确的，错误的会自动追加到 errlog中。虽然也可以在脚本中直接指定输出文件，但是，当要输出的较多时，则比较麻烦。

  • 永久重定向： exec 文件描述符>文件名 。对整个脚本的代码都有效

    #！ /bin/bash

exec 1>testoutput
exec 2>errlog

echo "Test Error" >&2 //把错误重定向到上边指定的文件中
echo "Nomal outPut"

  • 输入重定向 exec 0<文件名

#! /bin/bash
#从文件中读取，并输出
exec 0< errlog

count=1
while read line
do
    echo "第 $count行 :$line"
        count=$[$count +1]
done

管道

  • 将一个命令的输出，重定向至另一个命令的标准输出。command1 | comand2
  • ps -ef|grep java //将ps命令的输出，传递给grep进行快速的输出
  • ps -ef|less //将ps命令的输出，传递给less进行分页
  • ps -ef >tempfile、less<tempfile //不如管道方便
  • date | tee datefile ：tee命令有两个输出，一个是文件，一个是标准输出。执行后控制台上也会显示日期，datefile中也会有日期。

shell编程中的函数

  • 函数声明的两种方式如下：

    function name {
    commands
}

name(){
    commands
}

  • 下面是一个例子，定义了后直接调用（必须先定义后调用）：

    #! /bin/bash

function func1 {
    echo '第一种方式定义的函数'
}

func2(){
    echo '第二种方式定义的函数'
}

func1
func2

    echo “$?”

  • 函数默认的退出码echo "$?,执行成功时，状态码是0，不成功时，状态码是其他的值。
  • 函数的返回值使用 return 来实现。在函数的最后使用return 20,则状态码输入的就是20。退出的状态码必须在0到255之间，否则会报错。

  • 函数返回其他字符：

    value = `func1`
  echo $value

  • 函数参数

    #! /bin/bash  

add() {
    if [ $# -eq 2 ];then  //判断参数是否等于2个
        result=$[ $1+$2 ]
        echo $result
    else
        echo "Please input 2 params"
        return 1
    fi
}

value=`add $1 $2` //收集两个参数，传给函数add
if [ $? -eq 0 ]; then //如果函数返回值是0，说明执行成功
    echo $value
else
    echo "Err: $value"
fi

  • 函数中数组参数的处理

    #! /bin/bash
testarray(){
            echo "$@" //输出所有参数
            echo "$0" //shell本身的文件名
            echo "$0"
            echo "$#" //输出参数的数量
}

sum(){
    local result=0 //局部变量
    for var in $@
    do
        result=$[ $result + $var ]
    done
    echo $result
    local newarray=(`echo "$@"`)
    echo ${newarray[*]}
}

array=(1 2 3 4 5 6 7)

testarray ${array[*]}

testarray ${array[*]}
  • 全局变量：在整个脚本中都有效,在函数内使用的，不使用local修饰符，也是全局变量
  • 局部变量：仅在函数内部使用有效

  • 函数库。下面是一个引用函数库的例子：

    #! /bin/bash

source ./calc.sh

echo `add 12 23`

    也可以在命令行中，直接执行上面的两句。即：

    source ./calc.sh

echo `add 12 23
  • 在~/.bashrc这个文 件中，系统已加加载了众多的函数库
