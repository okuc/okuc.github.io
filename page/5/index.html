轻松在eclipse中将maven项目布署到tomcat上

在eclipse中导入已经建好的maven项目后，往往会出现无法布署到tomcat上的情况，此时，该怎么办呢？

  1. 右键项目名，选择Properties。在弹出的页面中，左侧选择Project Facets,在右侧的列表中，选中Dynamic Web Module、Java、JavaScript三项，然后点击右下角的Apply按扭。

  2. 左侧选择Deployment Assembly,在左则，配置好依赖：

    • /src/main/java–>WEB-INF/classes
    • /src/main/resources–>WEB-INF/classes
    • /src/main/webapp–>/
    • /src/test/java–>WEB-INF/classes

    • Maven Dependencies–>WEB-INF/lib

      然后点击右下角的Apply按扭。

  3. 点击OK按扭返回，这回，可以直接在项目名上点击右键，选择Run AS -> Run on Server了。

新手学习Angularjs之二：ng-bind、ng-model、大括号的区别

本文假设nodejs及bower环境变量已经设置完成，如果还没有设置，可参见这里：windows下搭建node.js及npm的工作环境（更新版）、Node.js的项目创建及依赖维护工作。

在学习的过程中，遇到了这三个对象，一时把我给搞糊涂了。经过百度，得到了以下答案：

  • ng-model：用于双向绑定，所以一般用于前台可输入的控件，如input。
  • ng-bind：只用于前台展示，所以一般和div结合使用。
  • 双大括号：其基本作用类似ng-bind，但不同的是，在angularjs未加载完成时，由于页面未解释，所以很有可能客户会看到大括号，所以一般<div ng-clock class="ng-clock">｛{uname}｝</div>这样使用，这样可使页面先不加载大括号，待全部加载完成后再解释执行。
